=>What are the differences between var, let and const ?

So var, let and const are keywords used to declare and initialize variables. Variables are basically container used to store any type of data. The basic difference between var, let and const is with var keyword we can re-declare and re-initialise same variable again and again. Ex-
var a=20;
var a=100;
var a=1000;

But with let we can declare a variable only once and not again and again but we re-assign the same variable again and again.Ex-
let a = 20;
a = 100;
a = 1000;

And in const we can declare and assign the value only once. We cannot re-declare and re-assign the variable. Also in const we have to initialize the variable and declare the variable in the same line. Ex-
const a = 100;

=>Explain the Pre-increment and Post-increment in JavaScript.

The increment(++) operator increments i.e adds one to its operands and returns the value before or after the increment, depending on where the operator is placed.

Post-increment--In this case, operator is placed after operand(x++), the increment operator increments and returns the value before incrementing. Ex-
let x = 3;
let y = x++;
console.log(x) //it will give output 4
console.log(y) //it will give output 3

Pre-increment--In this case, operator is placed before operand(++x), the increment operator increments and returns the value after incrementing. Ex-
let x = 3;
let y = ++x;
console.log(x) //it will give output 4
console.log(y) //it will give output 4.

=>Difference between "==" and "===" operators.

These are comparison operators. They are used to compare two operands.
In Javascript, the ‘==’ operator is also known as the loose equality operator which is mainly used to compare two values on both sides and then return true or false. Ex-
console.log(21 == 21); //true
console.log(21 == '21'); //true
console.log('food is love'=='food is love'); //true
console.log(21 == 32); //false
console.log(21 == '32'); //false

JavaScript ‘===’ operator, also known as strict equality operator, it compares both the value and the type which is why the name “strict equality”. Ex-
console.log('hello world' === 'hello world'); //true
console.log(true === true); //true
console.log(5 === 5); //true
console.log(true === 1); //false
console.log(true === 'true'); //false
console.log(5 === '5'); //false.
